# Server 模块 UI 结构

本文件定义 Server 模块的前端组件拆分方案、页面结构、导航设计、状态管理策略和交互模式。

参考主文档：[README.md](./README.md)

## 1. 设计原则

- 先选服务器，再进工作区
- 监控数据实时刷新，图表流畅
- 进程/服务列表支持搜索、排序、分页
- 日志支持流式滚动和关键词高亮
- 操作前二次确认，失败后支持重试

### 3.2 组件职责说明

| 组件 | 职责 | 依赖 |
|------|------|------|
| ServerSidebar | 展示所有服务器，支持状态筛选 | ServerItem, ServerStatus |
| ServerItem | 单个服务器的展示，包含状态和基本信息 | ServerStatus |
| ServerStatus | 服务器状态指示器，支持在线/降级/离线状态 | - |
| WorkspaceTabs | 工作区页签切换 | - |
| MonitorPanel | 监控页面主体，展示指标卡片和图表 | MetricsCard, MetricsChart |
| MetricsCard | 单个指标卡片（CPU/内存/磁盘/网络） | MetricsChart |
| MetricsChart | 指标趋势图表，支持时间范围切换 | - |
| ProcessList | 进程列表，支持搜索和排序 | ProcessRow |
| ProcessRow | 单个进程行 | - |
| ServiceList | 服务列表 | ServiceItem |
| LogViewer | 日志查看器，支持流式滚动和搜索 | LogEntry |

## 5. 交互模式

### 5.1 监控数据刷新

- 默认 5 秒自动刷新一次
- 可手动暂停刷新
- 图表数据按时间范围加载（1H/6H/24H/7D/30D）
- 鼠标悬停显示具体数值 tooltip

### 5.2 进程操作确认

```
┌──────────────────────────────────────┐
│           确认终止进程                │
├──────────────────────────────────────┤
│  您确定要终止进程 python3 (PID: 5678) │
│  吗？此操作将强制结束进程。           │
├──────────────────────────────────────┤
│  [取消]              [确认终止]      │
└──────────────────────────────────────┘
```

### 5.3 服务操作确认

```
┌──────────────────────────────────────┐
│           确认重启服务                │
├──────────────────────────────────────┤
│  您确定要重启 nginx.service 吗？      │
│  这将中断当前正在处理的请求。         │
├──────────────────────────────────────┤
│  [取消]              [确认重启]      │
└──────────────────────────────────────┘
```

### 5.4 操作状态反馈

- 操作进行中：显示 loading spinner
- 操作成功：显示绿色提示，3秒后自动消失
- 操作失败：显示红色提示，提供重试按钮
- 需要二次确认的操作显示确认对话框

## 6. 状态管理策略

### 6.1 状态分层

| 层级 | 作用域 | 存储位置 | 说明 |
|------|--------|----------|------|
| 全局 | 所有服务器 | serverStore | 服务器列表、连接状态 |
| 工作区 | 当前服务器 | serverStore | 当前选中服务器、当前页签、过滤条件 |
| 页面 | 当前视图 | useState/useQuery | 监控数据、进程列表、日志 |
| 组件 | 单个组件 | useState | 表单状态、展开折叠 |

### 6.2 关键状态

```ts
export type ServerWorkspaceState = {
  activeConnectionId: string | null;
  activeTab: ServerWorkspaceTab;
  monitorRange: "1h" | "6h" | "24h" | "7d" | "30d";
  processQuery: string;
  processSortBy: "cpu" | "memory" | "name" | "pid";
  processSortOrder: "asc" | "desc";
  logFilter: LogFilter;
  logFollow: boolean;
  selectedServiceName?: string | null;
};
```

### 6.3 数据获取策略

- 进入工作区时获取服务器信息和实时指标
- 监控数据按时间范围按需获取
- 进程/服务列表支持搜索防抖
- 日志支持流式订阅
- 操作成功后自动刷新相关数据

## 7. 路由设计

Server 模块采用嵌套路由结构：

```
/server                    # Server 模块入口
/server/connections        # 服务器管理（侧边栏）
/server/workspace          # 工作区
/server/workspace/monitor  # 监控页
/server/workspace/processes # 进程页
/server/workspace/services # 服务页
/server/workspace/logs     # 日志页
/server/workspace/files    # 文件页
/server/workspace/panel    # 面板资源页
```

## 8. 样式与主题

### 8.1 色彩规范

| 状态 | 颜色 | 用途 |
|------|------|------|
| 在线/正常 | 绿色 | 服务器在线、CPU/内存/磁盘正常 |
| 警告 | 黄色 | CPU/内存/磁盘使用率高、服务失败 |
| 严重/错误 | 红色 | CPU/内存/磁盘使用率极高、服务异常 |
| 离线 | 灰色 | 服务器离线 |

### 8.2 图表配色

| 指标 | 颜色 | 说明 |
|------|------|------|
| CPU | accent (蓝) | 主要指标色 |
| 内存 | warn (黄) | 内存使用 |
| 磁盘 | success (绿) | 磁盘使用 |
| 网络 | purple (紫) | 网络流量 |

## 9. 响应式设计

| 断点 | 布局变化 |
|------|----------|
| > 1280px | 三栏布局：侧边栏 + 工作区 |
| 768px - 1280px | 侧边栏折叠为图标模式 |
| < 768px | 侧边栏变为抽屉式，详情改为模态框 |

## 10. 性能优化策略

- 监控图表使用 Canvas 或 SVG 高效渲染
- 进程列表支持虚拟滚动（1000+ 项）
- 日志滚动使用虚拟列表
- 监控数据缓存
- WebSocket 复用（日志流）
- 防抖搜索

## 11. 可访问性

- 键盘导航支持
- 屏幕阅读器支持
- 高对比度模式
- 语义化 HTML 标签
- ARIA 标签
