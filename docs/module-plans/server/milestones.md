参考主文档：[README.md](./README.md)

| 阶段 | 版本 | 时间 | 核心目标 |

## 2. MVP 1: SSH 直连采集 + 基础监控 + 进程/服务

### 2.1 目标

建立 Server 模块的基础框架，实现 SSH 直连数据采集和基础管理功能。

### 2.2 时间安排

| 任务 | 时长 | 负责人 |
|------|------|--------|
| 连接模型与接口设计 | 1天 | 架构师 |
| SSH 直连采集服务 | 2天 | 后端开发 |
| 服务器列表前端组件 | 1天 | 前端开发 |
| 监控面板（CPU/内存/磁盘/网络） | 2天 | 前端开发 |
| 进程列表与搜索排序 | 2天 | 前后端协作 |
| 服务列表与状态查看 | 2天 | 前后端协作 |
| 测试与集成 | 2天 | 全组 |

### 2.3 交付物

**前端**:
- ServerPanel.tsx - 主入口组件（重构 mock 数据）
- ServerSidebar.tsx - 服务器侧边栏
- MonitorPanel.tsx - 监控面板
- MetricsCard.tsx - 指标卡片
- ProcessList.tsx - 进程列表
- ServiceList.tsx - 服务列表

**后端**:
- `/api/server/connections` - 连接管理接口
- `/api/server/metrics/snapshot` - 实时监控数据
- `/api/server/processes` - 进程列表接口
- `/api/server/services` - 服务列表接口
- SSH 命令执行服务

### 2.4 验收标准

1. ✅ 支持添加/编辑/删除服务器连接（SSH 直连）
2. ✅ 支持服务器状态实时显示（在线/离线/降级）
3. ✅ 支持 CPU/内存/磁盘/网络实时监控图表
4. ✅ 支持进程列表查看、搜索、排序
5. ✅ 支持服务列表查看、按状态筛选
6. ✅ 支持进程详情查看
7. ✅ 支持服务详情查看

### 2.5 技术要点

- SSH 命令执行封装
- 系统命令解析（top、free、df、ps 等）
- 监控数据定时采集
- WebSocket 实时推送（可选）

## 6. 质量标准

### 6.1 代码质量

- 代码覆盖率 ≥ 80%
- 无严重代码异味
- 符合团队代码规范

### 6.2 性能指标

- 页面加载时间 ≤ 2s
- 监控数据刷新 ≤ 1s
- 进程/服务列表渲染 ≤ 300ms（1000+ 项）
- API 响应时间 ≤ 500ms

### 6.3 安全标准

- 所有接口认证校验
- SSH 凭据加密存储
- 面板 API Key 加密存储
- SQL 注入防护
- XSS 攻击防护

### 6.4 用户体验

- 操作有反馈（loading/success/error）
- 支持快捷键操作
- 响应式设计支持移动端
- 操作失败支持重试

## 7. 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| SSH 连接不稳定 | 高 | 中 | 自动重连 + 状态缓存 |
| 系统命令输出格式不一致 | 中 | 中 | 多版本解析 + 容错处理 |
| 面板 API 版本兼容性 | 高 | 中 | 版本检测 + API 降级 |
| 大量历史数据查询性能 | 中 | 中 | 分页 + 缓存 + 预聚合 |
| 批量命令执行超时 | 中 | 中 | 超时配置 + 结果轮询 |
| 敏感日志泄露 | 低 | 高 | 脱敏处理 + 权限控制 |
