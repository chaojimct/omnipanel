# Docker 模块 UI 结构

本文件定义 Docker 模块的前端组件拆分方案、页面结构、导航设计、状态管理策略和交互模式。

参考主文档：[README.md](./README.md)

## 1. 设计原则

- 先选连接，再进工作区
- 能力决定展示，无能力则灰置或隐藏
- 状态变化需有过渡动画和加载反馈
- 长列表必须支持虚拟滚动和搜索过滤
- 动作前二次确认，失败后支持重试
- 默认界面保持“小而优”：先展示最关键状态、异常、日志和动作，不把 Docker 全量概念一次性堆给用户
- 所有危险动作必须显示影响范围，尤其是生产环境连接、删除、prune、kill、Compose down
- 容器详情中要保留“打开宿主机 SSH / 查看 Server 监控 / 发送给 AI 分析”的上下文入口

### 3.2 组件职责说明

| 组件 | 职责 | 依赖 |
|------|------|------|
| ConnectionList | 展示所有 Docker 连接，支持搜索筛选 | ConnectionItem, ConnectionStatus |
| ConnectionItem | 单个连接的展示，包含状态和基本信息 | ConnectionStatus |
| ConnectionStatus | 连接状态指示器，支持在线/降级/离线状态 | - |
| WorkspaceTabs | 工作区页签切换，根据能力决定显示哪些页签 | useDockerCapabilities |
| OverviewPanel | 总览页面，展示统计、异常和快捷操作 | StatsCard, AnomalyList, QuickActions |
| ComposeList | Compose 项目列表，支持过滤和搜索 | ComposeCard |
| ContainerList | 容器列表，支持按状态过滤 | ContainerCard |
| Terminal | 容器 exec 终端组件 | xterm.js + Tauri event |
| LogViewer | 日志查看器，支持滚动和搜索 | - |

## 5. 交互模式

### 5.1 动作确认模式

所有 destructive 动作必须二次确认：

```
┌──────────────────────────────────────┐
│           确认操作                    │
├──────────────────────────────────────┤
│  您确定要 [动作] [资源名称] 吗？       │
│                                      │
│  此操作 [后果说明]                    │
├──────────────────────────────────────┤
│  [取消]              [确认]          │
└──────────────────────────────────────┘
```

### 5.2 操作状态反馈

- 操作进行中：显示 loading spinner
- 操作成功：显示绿色提示，3秒后自动消失
- 操作失败：显示红色提示，提供重试按钮

### 5.3 能力降级处理

当连接能力不足时：
- 页签灰置并显示工具提示
- 按钮禁用并显示原因
- 状态消息说明能力限制

## 6. 状态管理策略

### 6.1 状态分层

| 层级 | 作用域 | 存储位置 | 说明 |
|------|--------|----------|------|
| 全局 | 所有连接 | dockerStore | 连接列表、连接状态 |
| 工作区 | 当前连接 | dockerStore | 当前选中连接、当前页签、过滤条件 |
| 页面 | 当前视图 | useState/useQuery | 列表数据、分页、搜索 |
| 组件 | 单个组件 | useState | 表单状态、展开折叠 |

### 6.2 关键状态

```ts
export type DockerWorkspaceState = {
  activeConnectionId: string | null;
  activeTab: DockerWorkspaceTab;
  filters: {
    composeStatus?: DockerProjectStatus;
    containerStatus?: DockerContainerLifecycle;
    imageFilter?: "all" | "dangling" | "used";
    searchQuery?: string;
  };
  selectedResources: {
    composeId?: string;
    containerId?: string;
    imageId?: string;
    volumeName?: string;
    networkId?: string;
  };
};
```

### 6.3 数据获取策略

- 进入工作区时获取连接信息和能力
- 切换页签时按需获取列表数据
- 列表数据采用缓存策略，切换页签时优先使用缓存
- 操作成功后自动刷新相关列表

## 7. 路由设计

Docker 模块采用嵌套路由结构：

```
/docker                    # Docker 模块入口
/docker/connections        # 连接管理（侧边栏）
/docker/workspace          # 工作区
/docker/workspace/overview # 总览
/docker/workspace/compose  # Compose 管理
/docker/workspace/containers # 容器管理
/docker/workspace/images   # 镜像管理
/docker/workspace/volumes  # 卷管理
/docker/workspace/networks # 网络管理
```

## 8. 样式与主题

### 8.1 色彩规范

| 状态 | 颜色 | 用途 |
|------|------|------|
| 在线 | 绿色 | 连接在线、容器运行中 |
| 降级 | 黄色 | 连接降级、容器警告 |
| 离线 | 红色 | 连接离线、容器异常 |
| 已停止 | 灰色 | 容器已停止 |

### 8.2 图标规范

- 使用 Lucide React 图标库
- 统一图标尺寸和间距
- 图标颜色与状态关联

## 9. 响应式设计

| 断点 | 布局变化 |
|------|----------|
| > 1280px | 三栏布局：侧边栏 + 工作区 |
| 768px - 1280px | 侧边栏折叠为图标模式 |
| < 768px | 侧边栏变为抽屉式，详情改为模态框 |

## 10. 性能优化策略

- 列表使用虚拟滚动（react-window 或 @tanstack/react-virtual）
- 图片懒加载
- 防抖搜索
- 缓存列表数据
- 按需加载组件
- Tauri event 监听按会话复用，离开页面及时取消日志和 exec 流

## 11. 可访问性

- 键盘导航支持
- 屏幕阅读器支持
- 高对比度模式
- 语义化 HTML 标签
