Design System v1.0

OmniPanel 设计语言 & 开发指南

终端原生、单字体、扁平深度。基于 OpenCode 设计系统构建的统一视觉语言，覆盖色彩、字体、组件、间距与开发规范。

Brand Identity

Logo

OmniPanel 标识由等宽字标 + 几何符号组成。终端光标「_」与方括号「[ ]」融合，象征命令行原生与面板化工作区。所有变体保持单色，不使用渐变。

主标识 · 暗色背景
单色变体 · 暗色背景
图标 · App / Favicon
反色标识 · 浅色背景
竖排组合
最小尺寸 24×24
Do

保持 logo 周围至少 1.5 倍图标宽度的留白区域。在深色背景上使用白色或蓝色变体。

Don't

不要对 logo 添加阴影、渐变、旋转或拉伸。不要在复杂背景上使用低对比度变体。

Color System

色彩系统

暖色调暗色主题，基于 rgb(32, 29, 29) 的近黑色背景。所有颜色均为语义化设计，不使用纯黑纯白。边框使用暖色透明 rgba(15, 0, 0, 0.12)。

Do

使用 CSS 变量引用颜色：var(--accent)。使用 color-mix() 派生透明度变体。

Don't

不要在组件中直接使用 hex 值。不要引入设计系统之外的新颜色。不要使用纯黑 #000 或纯白 #fff。

Typography

字体系统

全局使用 Berkeley Mono 等宽字体，无衬线/衬线变体。「一切皆代码」哲学 —— 层级通过字号和字重区分，而非字体切换。

标题、强调

Do

标题使用 font-weight: 700。正文使用 400 或 500。数字使用 font-variant-numeric: tabular-nums。

Don't

不要引入第二字体（Inter、Roboto 等）。不要在标题上使用 letter-spacing 负值以外的装饰。不要使用 300 以下的字重。

布局约束 Layout

最大内容宽度

--container: 1080px — 窄阅读列，单栏居中，聚焦注意力。

内边距

--gutter: 24px — 桌面端两侧留白。移动端缩减至 16px。

段落间距

Section 之间使用 clamp(48px, 8vw, 96px) 垂直间距，不使用装饰分隔线。

响应式断点

≤768px 单栏。≤920px 网格折叠。≤640px 隐藏导航、缩小终端字体。

Development Guidelines

开发规范

面向开发者的编码约定、组件用法、响应式策略和无障碍实践。

CSS Token 引用规范

/* ✅ 正确 — 使用 CSS 变量 */ .card { background: var(--surface); border: 1px solid var(--border); border-radius: var(--r-md); color: var(--fg); } /* ❌ 错误 — 直接使用 hex */ .card { background: #302c2c; /* 应该用 var(--surface) */ color: #fdfcfc; /* 应该用 var(--fg) */ } /* ✅ 使用 color-mix 派生透明度 */ .card:hover { background: color-mix(in oklch, var(--fg) 3%, var(--surface)); }

组件编写模式

<!-- ✅ 正确的按钮结构 --> <button class="btn btn-primary"> Download </button> <!-- ✅ 带图标的按钮 --> <button class="btn btn-secondary"> <svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.6"> <path d="M4 17l6-6-6-6M12 19h8"/> </svg> View on GitHub </button> <!-- ✅ 状态徽章 --> <span class="badge badge-success">Online</span> <span class="badge badge-danger">Error</span> <!-- ✅ 数据表格 --> <table class="spec-table"> <thead> <tr> <th>名称</th> <th>状态</th> </tr> </thead> <tbody> <tr> <td>nginx</td> <td><span class="badge badge-success">Running</span></td> </tr> </tbody> </table>

响应式策略 Responsive

断点 Breakpoints

≤640px 移动端 · ≤768px 平板 · ≤920px 网格折叠 · ≤1024px 笔记本 · ≥1440px 桌面

网格折叠

grid-2 / grid-3 / grid-4 在 ≤920px 时自动折叠为单栏。不要手动覆盖。

字体缩放

标题使用 clamp() 流式缩放。正文保持 16px 不变。终端字号 ≤640px 降至 12px。

触摸目标

按钮最小 28×28px。输入框 padding ≥6px。可点击区域不小于 44×44px（移动端）。

交互状态 Interaction States

状态 颜色变化 过渡时间 用途
Default --accent (#007aff) — 默认状态
Hover --accent-hover (#0056b3) 120ms 鼠标悬停
Active --accent-active (#004085) 即时 按下状态
Focus outline: 2px solid var(--accent) — 键盘焦点
Disabled --meta (#6e6e73) — 禁用状态，opacity 0.5

无障碍 Accessibility

对比度

主文字 #fdfcfc 在 #201d1d 上对比度 15.8:1，远超 WCAG AAA 标准。强调色 #007aff 在暗色背景上对比度 7.2:1。

焦点指示器

所有可交互元素必须有 :focus-visible 样式。使用 2px 蓝色 outline，offset 2px。

语义化 HTML

使用 <button> 而非 <div onclick>。使用 <nav> / <main> / <section> 语义标签。

图标可访问性

纯图标按钮必须有 aria-label。SVG 使用 stroke="currentColor" 继承文字颜色。

深度系统 Depth

OmniPanel 的深度系统是完全扁平的 —— 零 box-shadow。深度通过边框和背景色变化表达，与终端美学一致。

层级 处理方式 用途
Level 0 · Flat 无边框无阴影 默认元素
Level 1 · Subtle 1px solid var(--border) 卡片、输入框、分隔线
Level 2 · Tab 2px solid var(--muted) bottom Tab 活跃指示器
Level 3 · Outline 1px solid var(--border) + bg shift 浮层、弹窗

反模式 Anti-patterns

禁止

在组件中直接写 hex 颜色值

引入 Inter / Roboto 等第二字体

使用 box-shadow 创建深度

使用渐变背景（除非品牌要求）

使用通用 emoji 作为功能图标

在暗色背景上使用纯黑 #000

推荐

始终使用 var(--token) 引用颜色

全局 Berkeley Mono，层级靠 size/weight

使用 border + bg shift 表达深度

单色强调色，最多使用两次

使用 inline SVG 单线图标

使用暖色近黑 #201d1d 作为背景
